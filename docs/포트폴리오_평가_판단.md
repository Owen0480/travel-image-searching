# 이 프로젝트, 약한가? — 판단 요약

코드베이스(Spring, FastAPI, Frontend)를 기준으로 **솔직한 판단**만 정리했습니다.

---

## 결론: **약하지 않다. 중상 수준이다.**

- **취준/포트폴리오용**: 충분히 쓸 만하다.  
- **“내가 한 부분”을 명확히 쓰면** 더 잘 보인다.

---

## 1. 왜 약해 보일 수 있나

- **팀 프로젝트**라서 “내 기여”가 뭔지 안 보이면, 심사자가 전부 내가 한 걸로도, 반대로 별로 한 것 없어 보이게도 받아들일 수 있음.
- **제목만** “여행 추천”이면 단순 CRUD처럼 보일 수 있음.  
  → 실제로는 **이미지 유사도(CLIP) + RAG(Chroma) + LLM(Gemini) + 에이전트 그래프(LangGraph)** 가 들어가 있음.

그래서 **“뭘 어떻게 했는지”를 구체적으로 쓰는 게** 약해 보이지 않게 만드는 핵심이다.

---

## 2. 실제로 강한 부분 (코드 기준)

| 영역 | 수준 | 근거 (코드 기준) |
|------|------|------------------|
| **AI 파이프라인** | 높음 | CLIP 이미지 임베딩, **DB 이미지 사전 임베딩 캐싱**, 코사인 유사도 TopK, **임계값(0.6) 처리**, ChromaDB에 장소 문서 청킹·임베딩·`visit_area_id` 스코프 RAG, Gemini 가이드 생성, API 실패 시 한글 메시지 |
| **채팅 에이전트** | 높음 | LangGraph **StateGraph**, 의도 분류 → 일반/추천/무관 라우팅, **입력 충분성 체크**, 후보 생성·검증·재시도(상한 2회), 정보 보강·품질 검사·재시도, 옵션 필터·랭킹·최종 제안 등 **다단계 플로우** |
| **인증/보안** | 높음 | Google OAuth2, **JWT + Refresh Token**, **Redis 저장**, SecurityFilterChain, 경로별 permitAll/authenticated, **JwtFilter**, CORS·Credentials 처리 |
| **프론트 인증** | 높음 | Axios **인터셉터**: 요청 시 토큰 부착, **401/403 시 Refresh 호출 → 재시도**, 실패 시 로그인 페이지 리다이렉트 (무한 루프 방지 `_retry` 플래그) |
| **백엔드 구조** | 중상 | Spring: auth/user/recommend/chat/travelstyle/conversation 등 **도메인 분리**, WebClient로 FastAPI 연동, DTO·Repository·Exception 처리 |
| **프론트 구조** | 중상 | 이미지 검색(업로드·FormData·결과 카드·외부 링크), 채팅, 여행스타일, 로그인/콜백, 마이페이지, **공통 api + 인터셉터** |

정리하면, **토이 수준이 아니라** “설계된 파이프라인 + 인증/에이전트”가 들어간 **중규모 풀스택 + AI** 프로젝트에 가깝다.

---

## 3. 상대적으로 약한 부분 (보완하면 더 좋음)

| 항목 | 현재 | 보완 시 포트폴리오 플러스 |
|------|------|---------------------------|
| **테스트** | 단위/통합 테스트가 거의 없음 | 핵심 서비스 1~2개만이라도 테스트 코드 있으면 “품질 의식” 어필 |
| **배포/인프라** | 로컬 중심, docker-compose 수준 | “한 번이라도 AWS/클라우드에 올려봤다”만 있어도 차별화 |
| **문서** | README·보고서 있음 | “내가 한 부분” 1페이지만 정리해두면 면접에서 유리 |

---

## 4. 어떻게 보이는지 (심사자 입장 가정)

- **“여행 추천 사이트”만 보면** → 평범한 팀 프로젝트로 보일 수 있음.
- **“CLIP + Chroma RAG + Gemini + LangGraph, JWT/Refresh/Redis, Spring–FastAPI 연동”**까지 **키워드 + 내 역할**을 쓰면 → **중상급 포트폴리오**로 보일 가능성이 높음.

즉, **같은 프로젝트라도 “어떤 포인트를 어떻게 쓰느냐”에 따라 약해 보일 수도, 강해 보일 수도 있다.**

---

## 5. 한 줄 요약

- **기술적으로는 약하지 않다.**  
- **“내가 한 부분”을 구체적으로 쓰고, AI/인증/에이전트 키워드를 살리면** 포트폴리오로 충분히 쓸 만하다.

`docs/포트폴리오용_내_역할_정리.md`에서 본인 역할만 골라서 한 줄 요약 넣어 쓰면 된다.
